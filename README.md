# 歩行練習サポートアプリ

「行橋〜別府 100キロウォーク」に向けたウォーキング練習をサポートする iOS アプリです。

---

## できること

| 機能 | 内容 |
|---|---|
| ダッシュボード | iPhone の「ヘルスケア」と連携し、歩数・距離・消費カロリーをグラフと進捗バーで表示 |
| 周辺グルメ | 現在地の半径500m以内で「いま営業中」の店をワンタップで検索 |

---

## 画面の流れ

```
初回起動 ─→ オンボーディング（ヘルスケア連携の許可）─→ 本体（2タブ）
                                                        ├─ ダッシュボード
                                                        └─ グルメ
```

2回目以降はオンボーディングを飛ばして本体から始まります。

---

## つかいかた

### ダッシュボード
- 起動して連携を許可すると、当日の実績が自動で表示されます。
- 下に引っぱると最新データに更新できます。
- データが出ないときは「設定で許可を確認」から設定アプリで許可状態を見直せます。

### グルメ
- オンラインのとき、現在地まわりの営業中の店が距離の近い順に並びます。
- **100m以上歩くと**「この場所で更新」ボタンが出ます。押すとその場所で再検索します（自動では更新しません）。
- 下の「現在地で手動更新」でいつでも再検索できます。
- 電波が無いときは待機状態になり、**電波が戻ると自動で再表示**します。
- 営業時間は提供元データに基づく**目安**です（画面にも注記があります）。

## 開発者向け

- **動かすには**：Xcode で開き、`Info.plist` への利用目的の記載と HealthKit の有効化が必要です。
  手順は [SETUP.md](SETUP.md) を参照してください。
- **APIキー不要**：グルメはサンプルデータ（`MockPlacesService`）で動きます。
  本番は `GooglePlacesService` に差し替えます。
- **対応サイズ**：iPhone SE 〜 Pro Max、iPad、横向き、Split View まで全画面表示に対応する設計です。

### ファイル構成

```
walking_app/
├─ WalkingTrainingApp.swift   起動の入口（@main）
├─ AppRootView.swift          オンボーディング ⇄ 本体タブ / ダッシュボード
├─ OnboardingView.swift       ヘルスケア連携の許可フロー
├─ HealthKitManager.swift     ヘルスケア連携（読み取り専用）
├─ Gourmet/                   周辺グルメ機能一式
├─ Assets.xcassets/           アプリアイコン（AppIcon）
├─ icon_1024.png             アイコン原画（1024×1024・フルブリード）
└─ SETUP.md                   Xcode 側の必須設定
```

### アプリアイコンの適用（ホーム画面表示用）
1. Xcode プロジェクトに `Assets.xcassets` を取り込む（既存の場合は `AppIcon.appiconset` を差し替え）。
2. ターゲット → General → App Icons → Source を `AppIcon` に設定。
3. ビルドすると、ホーム画面に歩行アイコンが表示されます（角丸は iOS が自動付与）。

> アイコンは透過なし・角丸/白枠なしのフルブリード正方形（`icon_1024.png`）です。
> 差し替える場合も同じ仕様の 1024×1024 PNG を `AppIcon-1024.png` に上書きしてください。

## 設計上のこだわり

- **オフラインに強い**：電波の無い山間部でも実績表示は動作し、グルメは復帰時に自動再取得。
- **省電力・低通信**：常時測位はせず100m移動時のみ検知。店舗画像は遅延読み込みで通信を節約。
- **大きく押しやすいUI**：歩行中・疲労時でも誤操作しにくいボタン設計。
